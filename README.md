## 网站结构 ##



- 网站的后端是用django写的，是python很火的一个web框架

- 数据库使用的是mysql，

- 服务器是借同学的，在搬瓦工上用支付宝买的最便宜的，位于美国加利福尼亚，正式环境使用的nginx+uwsgi+django，静态文件是通过nginx处理的，开发环境使用的只有django,

- 代码管理使用的git，如果后端要更新代码的话，是先从本地push推送到github，服务器这边是用Xshell5远程操作centos将代码pull拉取下来，然后重启uwsgi，

- 数据则是通过navicat远程连接服务器数据库进行数据传输更新数据，

- 博客现在是用markdown语法写的，然后后端通过python的markdown库转化为html返回给前端,说实话这种做法并不好，因为后端是应该只提供数据的，根据数据创建dom元素应该是前端的事情，这样也方便添加一些样式和事件，未来可能改成后端提供一个article数组，数组里每个对象都有一个标签属性，文本属性，前端根据这些数据创建dom元素

- 网站的前端开发环境是通过webpack搭建的，

- 开发框架选用的是vue，

- UI风格使用的是bootstrap，

- 还用到了jquery，

- 图标有些是自己做的,有些是用awesome图标，

- css采用了scss语法,

- 现在前端页面结构是一个vue的根组件连接其它vue组件,一个vue组件就是一个.vue文件,最后这些组件，样式，js文件都会通过webpack打包成一个js文件.

- 网站是前后端分离的，前后端数据的交互是通过ajax请求

- **blogproject文件夹放的是django后端项目,**

- **blogweb文件夹里放的是webpack前端项目**
